Leetcode Notes——Array and Matrix

记录一下自己 leetcode 的刷题记录，本部分为数组与矩阵篇。部分内容搬运自 Leetcode-cn 大佬题解、wind-liang 题解、chenBright 题解。

283. 移动零[easy]

解法一：双指针（交换）——一次遍历

参考快速排序的思想，快速排序首先要确定一个待分割的元素做中间点x，然后把所有小于等于x的元素放到x的左边，大于x的元素放到其右边。
这里可以用0当做这个中间点，把不等于0（注意题目没说不能有负数）的放到中间点的左边，等于0的放到其右边。这里的中间点就是0本身，所以实现比快速排序简单，使用两个指针i和j，只要nums[i]!=0，就交换nums[i]和nums[j]。

复杂度分析：

  • 时间复杂度：O(n)。
  • 空间复杂度：O(1)。

解法二：双指针（不交换）——两次遍历

其实可以不需要交换，只要将非0元素移动到数组前面，再在后面补0即可。

创建两个指针i和j，第一次遍历的时候指针j用来记录当前有多少非0元素。即遍历的时候每遇到一个非0元素就将其往数组左边挪，第一次遍历完后，j指针的下标就指向了最后一个非0元素下标。第二次遍历的时候，起始位置就从j开始到结束，将剩下的这段区域内的元素全部置为0。

该实现的最坏情况是：存在大量的0元素，例如[0 0 0 ... 0 1]。最后需要补n - 1个0。但是基于交换的实现，只需要交换一次即可，不需要其他操作。

复杂度分析：

  • 时间复杂度：O(n)。
  • 空间复杂度：O(1)。

566. 重塑矩阵[easy]

解法一：借助队列

借助队列来放置按行提取的元素。然后，取出以串行顺序形成的队列元素，并再次按行逐行排列所得到的所需矩阵中的元素。如果原始矩阵中的元素数量不等于所得矩阵中的元素数量，则不可能形成所得矩阵，直接返回原始矩阵。

复杂度分析：

  • 时间复杂度：O(m*n)。遍历 m∗n 元素两次。m 和 n 分别表示给定矩阵的行数和列数。
  • 空间复杂度：O(m*n）。形成的队列大小为 m∗n 。

解法二：不用额外空间

逐行顺序迭代

不使用队列，而是在逐行顺序迭代给定矩阵的同时，直接将数字放在结果矩阵中。在将数字放入结果数组时，固定一个特定的行，并继续增加列数，直到到达 c 指示的所需列的末尾。此时，通过递增来更新行索引，并将列索引重置为从 0 开始。因此，可以节省队列消耗的空间。

除法和取模

在上面的写法中，需要跟踪何时到达结果矩阵的列的末尾，并且需要通过每次检查当前索引来更新当前行和列号以放置提取的元素。实际可以利用数学运算来帮助解决，而不是在每一步都进行限制性检查。

二维数组在内存中存储时表示为一维数组，二维数组nums[m][n]在存储元素nums[i][j]时实际索引到nums[n*i+j]。而我们需要以相反的顺序处理。因此设置一个变量count来统计已放置的元素个数，res[count/c][count%c]即为当前元素应放置的位置，c为reshape后的列数。

复杂度分析：

  • 时间复杂度：O(m*n)。遍历 m∗n 个元素。m 和 n 分别表示给定矩阵的行数和列数。
  • 空间复杂度：O(1)。

485. 最大连续1的个数[easy]

解法一：数组遍历

遍历数组，记录连续 1 的个数，比较是否是当前最大个数。

解法二：滑动窗口

当输出或比较的结果在原数据结构中是连续排列的时候，可以使用滑动窗口算法求解。

将两个指针比作一个窗口，通过移动指针的位置改变窗口的大小，观察窗口中的元素是否符合题意。

  • 初始窗口中只有数组开头一个元素。
  • 当窗口中所有元素为 1 时，右指针向右移，扩大窗口。
  • 当窗口中存在 0 时，计算连续序列长度，左指针指向右指针。

74. 搜索二维矩阵[medium]

解法一：二分查找

输入的 m x n 矩阵可以视为长度为 m x n的有序数组，即可将二维矩阵的搜索转化为一维数组的二分查找问题。

该虚数组的索引与二维矩阵中的行和列的转化规则为： row = idx / n 、 col = idx % n 。

复杂度分析：

  • 时间复杂度 : 标准的二分查找，时间复杂度为 O(log(mn))，即 O(logm + logn)。
  • 空间复杂度 : O(1)。

解法二：从右上角向左下角查找

从右上角向左下角查找：

  1. 如果matrix[row][col] == target，则查找成功。
  2. 如果matrix[row][col] > target，则继续向左查找，即列左移col--。
  3. 如果matrix[row][col] < target，则继续向下查找，即行下移row++。

如果结束都没有查找成功，则查找失败。

也可以从左下角向右上角查找。

复杂度分析：

  • 时间复杂度 : O(m+n)。
  • 空间复杂度 : O(1)。

240. 搜索二维矩阵 II[medium]

解法一：二分查找

逐行查找

一行一行的进行二分查找，结合有序的性质，一些情况可以提前结束。

  • 某一行的第一个元素大于了target，当前行和后边的所有行都不用考虑了，直接返回false。
  • 某一行的最后一个元素小于了target，当前行就不用考虑了，换下一行。

复杂度分析：

  • 时间复杂度 : O(mlog(n))。
  • 空间复杂度 : O(1)。

以对角线上的元素为起点，对行和列进行二分查找

复杂度分析：

  • 时间复杂度 : O(log(n!))。
  • 空间复杂度 : O(1)。

对数组分块进行二分查找

复杂度分析：

  • 时间复杂度 : O(nlogn)。
  • 空间复杂度 : O(1)。

解法二：从右上角向左下角查找

思路与 74. 搜索二维矩阵 解法二相同，也可以从左下角向右上角查找。

复杂度分析：

  • 时间复杂度 : O(m+n)。
  • 空间复杂度 : O(1)。

378. 有序矩阵中第K小的元素[medium]

解法一：直接排序

将这个二维数组另存为一维数组，并对该一维数组进行排序。最后这个一维数组中的第 k 个数即为答案。

复杂度分析：

  • 时间复杂度：O(n^2logn)，对 n^2 个数排序。
  • 空间复杂度：O(n^2)，一维数组需要存储这 n^2 个数。

解法二：归并排序

矩阵的每一行均为一个有序数组。问题即转化为从这 n 个有序数组中找第 k 小的数，可以利用归并排序的做法，归并到第 k 个数即可停止。一般归并排序是两个数组归并，而本题是 n 个数组归并，所以需要用小根堆维护，以优化时间复杂度。

复杂度分析：

  • 时间复杂度：O(klogn)，归并 k 次，每次堆中插入和弹出的操作时间复杂度均为 logn。
  • 空间复杂度：O(n)，堆的大小始终为 n。

解法三：二分查找

  1. 找出二维矩阵中最小的数 left，最大的数 right，那么第 k 小的数必定在 left~right 之间；
  2. mid=(left+right) / 2；在二维矩阵中寻找小于等于 mid 的元素个数 count；
    • 若这个count 小于 k，表明第 k 小的数在右半部分且不包含 mid，即 left=mid+1, right=right，又保证了第 k 小的数在 left~right 之间。
    • 若这个count 大于 k，表明第 k 小的数在左半部分且可能包含 mid，即 left=left, right=mid，又保证了第 k 小的数在 left~right 之间。
  3. 因为每次循环中都保证了第 k 小的数在 left~right 之间，当 left==right 时，第 k 小的数即被找出，等于 left。

注意：这里的 left、mid、right 是数值，不是索引位置。

取 mid=8 时寻找小于等于 mid 的元素个数示意图：

复杂度分析：

  • 时间复杂度：O(nlog(r−l))，二分查找进行次数为 O(log(r−l))，每次操作时间复杂度为 O(n)。
  • 空间复杂度：O(1)。

645. 错误的集合[easy]

解法一：暴力法

单独检查 1 到 n 的所有数字。在检查每个数字时都遍历整个 nums 数组，检查当前数字在 nums 中是否出现了两次，或者一次都没有出现。使用 dup 和 missing 记录重复数字和缺失数字。一旦找到这两个数字，就提前结束查找过程。

C++超时，Java不超时。

复杂度分析：

  • 时间复杂度：O(n^2)。在最坏的情况下，需要在 1 到 n 的每个数字上遍历一次 nums。
  • 空间复杂度：O(1)。

解法二：使用map

使用哈希表统计数组中元素出现的次数，再检查 1 到 n 的每个数字在哈希表中出现次数，即可找出缺失数字和重复数字。

复杂度分析：

  • 时间复杂度：O(n)。遍历 nums 需要时间 O(n)，在 map 中检查每个数字需要时间 O(n)。
  • 空间复杂度：O(n)，map 需要存储数组元素。

解法三：利用数组索引

通过不断交换数组元素，让每个元素nums[i]放在索引位置nums[i]-1上，数组变成[1,2,3,4,5...n]。然后通过判断nums[i]!=i+1找到重复元素和缺失元素。

复杂度分析：

  • 时间复杂度：O(n)。
  • 空间复杂度：O(1)。

解法四：异或

需要一个 [1, 2, …n] 的数组作为辅助，假设重复元素为 dup，丢失元素为 mis。将原数组和辅助数组所有元素进行异或，可知其中等于 dup 的有3个，mis有1个（在辅助数组中），其余元素各有2个。在异或运算过程中，成对出现的都会抵消为 0。最后的结果就是 dup^mis。接下来问题就转化为如何将 dup 和 mis 分开，参考 算法笔记 中题目“一个数组中有两种数出现了奇数次，其他数都出现了偶数次，怎么找到这两个数”思路即可。

复杂度分析：

  • 时间复杂度：O(n)。
  • 空间复杂度：O(1)。

697. 数组的度[easy]

解法：利用哈希表

  • 具有度数 d 的数组必须有一些元素 x 出现 d 次。如果某些子数组具有相同的度数，那么某些元素 x （出现 d 次）。最短的子数组是将从 x 的第一次出现到最后一次出现的数组。
  • 对于给定数组中的每个元素， left 是它第一次出现的索引； right 是它最后一次出现的索引。例如，当 nums = [1,2,3,2,5] 时，left[2] = 1 和 right[2] = 3。
  • 然后，对于出现次数最多的每个元素 x，right[x] - left[x] + 1 将是候选答案，取这些候选的最小值。

复杂度分析

  • 时间复杂度：O(N)。其中 N 是 nums 的长度。
  • 空间复杂度：O(N)，first，count 使用的空间。

766. 托普利茨矩阵[easy]

解法一：对角线法

首先需要知道怎么判断 (r1, c1 和 (r2, c2) 这两个点属于一条对角线。通过观察可以发现，在满足 r1 - c1 == r2 - c2的情况下，这两个点属于同一条对角线。

利用哈希表存储每条对角线上遇到的第一个元素的值，如果之后遇到的任何一个值不等于之前存储的值，那么这个矩阵就不是托普利茨矩阵，否则就是。

复杂度分析

  • 时间复杂度: O(M*N)，即矩阵大小。
  • 空间复杂度: O(M+N)。

解法二：检查左上邻居

对于每条对角线上的元素，按顺序有 a1,a2,a3,…ak。如果所有对角线上的元素都满足 a1=a2, a2 = a3, a{k-1} = ak，那么这个矩阵就是托普利茨矩阵。对于对角线上的元素来说，如果当前元素不是第一个出现的元素，那么它前面的元素一定在当前元素的左上角，因此只需要检查只需要检查 matrix[r-1][c-1] == matrix[r][c]就可以了。

复杂度分析

  • 时间复杂度: O(M*N)。
  • 空间复杂度: O(1)。

350. 两个数组的交集 II[easy]

解法一：排序+双指针

如果两个数组是有序的，则可以使用双指针的方法得到两个数组的交集。

时间复杂度 O(mlogm+nlogn)，空间复杂度 O(1)。

解法二：哈希计数

将较小的数组哈希计数，随后在另一个数组中根据哈希来寻找。

时间复杂度 O(m+n)，空间复杂度 O(min(m, n))。

如果 nums2 的元素存储在磁盘上，磁盘内存是有限的，并且你不能一次加载所有的元素到内存中。那么就无法高效地对 nums2 进行排序，因此推荐使用哈希计数而不是排序+双指针。在哈希计数中，nums2 只关系到查询操作，因此每次读取 nums2 中的一部分数据，并进行处理即可。

1502. 判断能否形成等差数列[easy]

解法一：排序+模拟

时间复杂度 O(NlogN)，空间复杂度 O(1)。

解法二：两次遍历

第一次遍历：

  • 得出最大值，最小值
  • 把每个元素都添加到集合
  • 遍历结束后，计算最大值和最小值的差，然后用这个差除以 n-1，就可以得到等差（如有）

第二次遍历：

  • 从最小值开始，每次加上上面求出的等差，判断和是否全在集合中
  • 如果发现有任意不在集合的就返回 False，全在返回 True

时间复杂度 O(N)，空间复杂度 O(N)。
